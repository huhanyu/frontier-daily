Agent 技能记忆

D2Skill: Dynamic Dual-Granularity Skill Bank for Agentic RL

一句话：把 Agent 经验拆成"任务级技能"+"步骤级技能"，用 RL 同时训练策略和技能库——在 ALFWorld/WebShop 上比无技能基线高出 10-20 个百分点。

核心思路：现有技能方法主要从轨迹级提取经验，缺乏精细的技能维护机制。D2Skill 把技能库分成两层：task skills 提供高层任务指引，step skills 提供细粒度纠错支持。关键机制是"配对 rollout"——同一策略下做 skill-free 和 skill-injected 两次执行，用性能差距推导 hindsight utility 信号，同时驱动技能更新和策略优化。技能库从训练经验中持续构建，通过 reflection 扩展、utility-aware 检索和剪枝维护。
启发：记忆不只是"存什么"，更是"怎么分层存、怎么动态维护"。双粒度设计让技能库同时服务规划和执行两个层面，比单一粒度更有弹性。

RL 算法进化

Evolutionary Discovery of Reinforcement Learning Algorithms via Large Language Models

一句话：用 LLM 做进化算子，直接在"可执行的 RL 更新规则"空间中搜索——发现的新 RL 算法在 Gymnasium 上可与 SAC/PPO/DQN/A2C 匹敌。

核心思路：RL 算法的更新规则通常是手工设计的（如 PPO 的 clipped surrogate）。本工作基于 REvolve 框架，把 LLM 作为 generative variation operator，在可执行更新规则的空间中做进化搜索。刻意排除 actor-critic、TD loss、value bootstrapping 等经典机制，迫使搜索发现非标准学习规则。发现的算法通过 post-evolution refinement（LLM 提议超参范围）进一步调优，端到端在 Gymnasium 上评估。
启发："用 Agent 设计 Agent"——LLM 不只是被优化的对象，也可以成为优化算法本身的发现者。这种 meta-level 进化可能解锁人类研究者想不到的 RL 更新策略。

上下文工程

ACE: Agentic Context Engineering — Evolving Contexts for Self-Improving Language Models

一句话：把 Agent 上下文（system prompt、memory）当作可进化的 playbook——通过模块化的 generation → reflection → curation 流程持续积累和精炼策略，在 Agent 基准上 +10.6%，金融推理上 +8.6%。

核心思路：现有上下文优化方法有两大问题：brevity bias（详细洞察被压缩为简短总结）和 context collapse（迭代改写导致细节丢失）。ACE 将上下文视为"不断进化的 playbook"，通过模块化的 generation → reflection → curation 流程进行增量更新，保留详细知识，同时适应长上下文模型。既可离线优化 system prompt，也可在线管理 agent memory。
启发：上下文工程不只是"写好 prompt"，而是让 prompt/记忆自身具备演化能力。这可能是 Agent 长期运行后保持效果的关键机制——记忆不会因为重复改写而退化。
