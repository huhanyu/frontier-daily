涌现自组织

Drop the Hierarchy and Roles: How Self-Organizing LLM Agents Outperform Designed Structures

一句话：25,000 个任务的实验显示，给 LLM Agent 一个协议（而非预设角色），它们会自发涌现角色分工和浅层层级——自组织比中心化协调高出 14%，8 个 Agent 能产生 5,006 个独特角色。

核心思路：跨越 8 个模型、4–256 个 Agent、8 种协调协议的大规模实验。关键发现：(1) Agent 在极简结构（固定顺序）下就能自发发明专业角色、主动放弃不擅长的任务、形成层级，无需任何预设。(2) Sequential 混合协议比集中式协调高 14%（p<0.001），协议间效果差距达 44%（Cohen's d=1.86）。(3) 自组织能力随模型能力缩放——强模型能自组织，弱模型仍需刚性结构。(4) 系统在 256 个 Agent 时质量无衰减（p=0.61），开源模型达到闭源 95% 的质量但成本低 24 倍。
启发：Multi-Agent 系统的设计范式可能需要翻转——不是"人类设计角色让 Agent 执行"，而是"给 Agent 协议和能力，让角色自己涌现"。随着模型变强，后者的收益会越来越大。

服务对话自进化

SEAD: Self-Evolving Agent for Multi-Turn Service Dialogue

一句话：用自进化替代大规模人工标注——SEAD 将用户建模拆成 Profile Controller（控制课程多样性）和 User Role-play Model（真实角色扮演），让对话 Agent 在模拟中自我迭代，任务完成率提升 17.6%，对话效率提升 11.1%。

核心思路：服务对话的难点在于高质量标注数据稀缺，且模拟用户行为容易变成不公平对抗——模拟器可以任意控制结果。SEAD 的解法：把用户建模拆成两部分——Profile Controller 生成多样的用户状态来管理训练课程，User Role-play Model 专注于逼真的角色扮演。这个设计保证环境提供自适应训练场景，而不是一个可以操控一切的对手。不需要大规模人工标注，Agent 在自博弈中持续进化策略。实验在任务完成率上超越开源和闭源基线 17.6%。
启发："自进化"不一定要在 RL 算法层面做文章。SEAD 的思路是在环境/用户模拟层面引入进化机制——通过控制训练课程的多样性来驱动策略提升，本质上是一种 curriculum learning + self-play 的融合。

多 Agent 科研进化

Mimosa: Toward Evolving Multi-Agent Systems for Scientific Research

一句话：让多 Agent 科研系统的工作流自动合成和迭代——Mimosa 通过 MCP 动态发现工具、meta-orchestrator 生成工作流拓扑、LLM judge 评分驱动迭代，在 ScienceAgentBench 上达到 43.1% 成功率，超越单 Agent 和静态多 Agent 方案。

核心思路：现有 ASR 系统受限于固定工作流和工具集，无法适应演化中的任务。Mimosa 的架构：(1) 通过 MCP（Model Context Protocol）动态发现可用工具；(2) meta-orchestrator 自动生成工作流拓扑；(3) 代码生成 Agent 执行子任务并调用科学软件；(4) LLM judge 对执行评分，反馈驱动工作流迭代优化。在 DeepSeek-V3.2 上达到 43.1% 成功率。一个有意思的发现：不同模型对多 Agent 分解和迭代学习的响应差异很大，说明工作流进化的好处取决于底层执行模型的能力。完全开源，执行日志完整可审计。
启发："工作流进化"是一个很实用的范式——不只是 Agent 在进化，Agent 之间的协作拓扑也在进化。MCP 作为工具发现层的设计值得关注，它把"发现工具"从人工配置变成了自动化流程。
