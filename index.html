多层记忆架构

Multi-Layered Memory Architectures for LLM Agents

一句话：提出 LOCOMO 三层记忆架构——在长上下文任务中分别维护情景、语义和工作记忆，在 LOCOMO、LOCCO、LoCoMo 三个 benchmark 上达到 46.85% 成功率，显著超越基线。

核心思路：长上下文任务中，LLM 容易"遗忘"早期关键信息。传统方法要么增加 context window，要么用 RAG 但效果有限。LOCOMO 的创新在于引入三层记忆：(1) Episodic Memory 记录最近交互，(2) Semantic Memory 存储任务相关的长期知识，(3) Working Memory 维护当前任务状态。关键设计：三层之间通过注意力机制动态交互，让模型在不同任务阶段调用不同层次的记忆。实验在三个专门设计的长上下文 benchmark（LOCOMO、LOCCO、LoCoMo）上验证了分层设计的有效性，达到 46.85% 成功率。
启发：这篇工作提示我们，长上下文模型的"大海捞针"能力可能不等于有效利用——需要显式的记忆管理机制来指导模型何时检索什么层次的信息。

极简推理

Yet Even Less Is Even Better For Agentic, Reasoning, and Coding Tasks

一句话：Less is More 系列的最新进展——系统性验证在 Agent、Reasoning、Coding 任务上，极简 prompt 和推理策略优于复杂方案，在多个 benchmark 上达到 SOTA。

核心思路：这篇是 Less is More 系列的第三篇（继 Se-agent 和 EAgent 之后）。核心发现：复杂 prompt 带来的边际收益递减，有时甚至有害。研究者系统性地比较了：(1) 极简 system prompt vs 冗长指令，(2) 少样本 vs 零样本，(3) 单步推理 vs 多步分解。结论：在 Agent、Reasoning、Coding 三个领域，极简方案普遍优于复杂方案，特别是在开源模型上效果显著。消融实验揭示了关键因素：清晰的任务边界比丰富的示例更重要。
启发：这波"减法"浪潮提示我们，prompt 工程的最佳实践可能不是"告诉模型更多"，而是"让模型自己推理更多"。随着模型能力提升，简约 prompt 的效果会越来越好。
