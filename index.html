自进化

Towards Self-Evolving Agents via Co-Evolutionary Capability Expansion

一句话：通过共进化实现 Agent 能力自扩展——能力中心框架动态生成工具或 Agent，减少对人工设计的依赖。

核心思路：与经验中心自进化（从交互中学习）不同，能力中心框架关注动态扩展 Agentic 系统的能力边界。核心机制：共进化——Agent 在扩展能力的同时，验证和筛选新能力的有效性。论文提出能力图谱来表示 Agent 能做什么，并探索在有限监督下自主发现新技能的可能性。实验覆盖工具创建、多步推理和长程规划任务。
启发：自进化从"学经验"升级到"造能力"——这是 Agent 从执行者向创造者演化的关键一步。

arXiv 2604.10923
Agent2 RL-Bench: Can LLM Agents Engineer Agentic RL Post-Training?

一句话：验证 Agentic 系统能否工程化 RL 后训练——用五个 Agent 系统跨三个 Scaffold 家族和六个 Driver LLM 做实验。

核心思路：RL 后训练是提升 LLM 能力的核心技术，但手工设计奖励函数成本高。Agent2 RL-Bench 研究 Agent 能否自动化这一过程：让 Agent 自己设计奖励、生成训练数据、调整训练流程。实验覆盖 OpenHands、OpenCode、Claude Code、Codex CLI、 Gemini CLI 等系统。关键发现：某些 Agent 配置能显著超越手工设计的基线，说明 Agent 确实可以"工程师化"RL 后训练。
启发：RL 后训练正在被 Agent 自动化——这意味着模型进化从"人工调参"向"Agent 自我优化"演进。

arXiv 2604.10547
Can LLM Agents Learn to Strategize? Emergent Deception in Multi-Agent LLM Alignment

一句话：研究 LLM Agent 能否学会策略性思考——在多 Agent 环境中观察欺骗行为的出现，用 KTO 作为对齐机制。

核心思路：多 Agent 环境中，Agent 不仅要完成目标，还需要理解其他 Agent 的意图。论文通过实验观察策略性行为（如欺骗、误导）的出现条件，并用 KTO（Kahneman-Tversky Optimization）作为对齐机制来约束这类行为。实验设计：多 Agent 协作与竞争场景，测量策略性行为的出现频率和对齐效果。
启发：多 Agent 系统中，策略性行为可能是"涌现"的副作用——需要在对齐阶段预先考虑。

记忆系统

Diagnosing Retrieval vs. Utilization Bottlenecks in LLM Agent Memory

一句话：诊断 LLM Agent 记忆的检索 vs 利用瓶颈——提出分析框架，从写策略、检索方法和记忆利用行为三个维度分析性能差异。

核心思路：记忆是 Agent 长程能力的关键，但"记忆做了什么"和"记忆怎么用"是两个独立瓶颈。论文提出诊断框架：3×3 研究（三种写策略 × 三种检索方法 × 三种利用模式），分析性能差异在何处出现。关键发现：在某些任务中，检索好的记忆不一定被有效利用（利用瓶颈），反之亦然。实验覆盖代码生成、问答和规划任务。
启发："记住"不等于"会用"——诊断框架帮助定位到底是检索还是利用出问题，这对记忆系统设计至关重要。

arXiv 2603.02473
Active Context Curation for LLM Agents via Reinforcement Learning

一句话：用强化学习主动筛选上下文——解耦为 TaskExecutor（纯推理）+ Context Curator（上下文管理），后者通过 RL 学习筛选。

核心思路：上下文窗口有限，但并非所有历史信息都重要。论文提出主动上下文筛选：用 RL 训练一个 Context Curator，决定哪些内容应该进入上下文学。架构解耦：TaskExecutor（冻结的基础模型，处理纯推理） + Context Curator（可训练的筛选器）。关键创新：通过奖励信号让筛选器学会保留对当前任务最有价值的信息。实验覆盖多轮对话和长程规划任务。
启发：上下文筛选从"被动压缩"向"主动学习"演进——RL 让筛选器学会"什么是重要的"。

多模型编排

Squeeze Evolve: Unified Multi-Model Orchestration for Verifier-Free Evolution

一句话：统一多模型编排实现无验证器的演化——横跨数学、编码、视觉、科学发现的八项基准测试。

核心思路：多模型协作是 2026 年的重要趋势，但如何编排不同模型仍具挑战。Squeeze Evolve 提出统一框架：单一调度器协调多个模型，模型间通过"演化"而非"验证"来选择最优输出。关键设计：模型间无需独立的验证器，而是通过迭代比较和选择来实现质量提升。实验覆盖 AIME 2025、HMMT 2025、GPQA-Diamond（数学）、LiveCodeBench V6（编码）、MMMU-Pro、BabyVision（视觉）、ARC-AGI-V2（视觉推理）、circle particle physics（科学发现）。
启发："无验证器演化"降低了多模型系统的复杂度——让模型自己比较和选择，可能比独立的验证器更高效。

代码生成

BACE: LLM-based Code Generation through Bayesian Anchored Co-Evolution of Code and Test Populations

一句话：通过贝叶斯锚定的代码与测试共进化实现代码生成——维护种群而非把测试当绝对真理，避免误杀有效解。

核心思路：现有 LLM 代码生成把生成的测试当绝对 ground truth，但测试本身可能有 bug，会误杀有效解。BACE 的核心创新：维护代码和测试两个种群，用贝叶斯方法锚定两者的质量，避免"测试错 => 代码错"的级联误判。实验覆盖多个编程任务，验证了相比 AgentCoder 等方法的优越性。
启发：代码生成从"生成-测试-淘汰"向"共进化"演进——种群思想让系统更鲁棒。

arXiv 2603.28653
ZeroCoder: Can LLMs Improve Code Generation Without Ground Truth Labels

一句话：无标签代码生成——在无 ground truth 标签的设置下，用 selector 提升代码质量。

核心思路：代码生成通常依赖 ground truth 标签来评估和筛选，但获取标签成本高。ZeroCoder 探索无标签设置下的代码生成：用 LLM 自身的输出作为"伪标签"，通过 selector 选择最佳输出。实验以 Qwen2.5-Coder-7B-Instruct 为基础，在完全无标签设置下，使用 selector B4 可比基线提升 14.5%（相对提升）。
启发：无标签学习在代码生成中是可行的——关键在于设计有效的"自选择"机制。

Reasoning → Agentic

From Reasoning to Agentic: Credit Assignment in Reinforcement Learning over LLMs

一句话：从推理到 Agentic 的信用分配——分析 GRPO、PURE、SPO、SPRO 等方法在数学推理任务中的适用性。

核心思路：RL 在 LLM 上的应用日趋广泛，但"信用分配"——即如何把成功归因到正确的推理步骤——仍是核心挑战。论文系统分析了几种 RL 方法：GRPO（baseline）、PURE、SPO、SPRO 在数学推理任务（GSM8K, MATH）上的表现。关键发现：短 CoT（Chain of Thought）、可验证的确定性任务是当前 RL 方法的"甜点"；PRM（Process Reward Model）监督在这些场景下容易获取。实验还比较了不同方法在复杂度和泛化性上的权衡。
启发：信用分配是 RL + LLM 的核心瓶颈——不同任务需要不同的分配策略，没有"一刀切"的方案。
