记忆系统 多智能体

记忆系统

Memory Intelligence Agent

一句话：深度研究智能体（DRA）集成 LLM 推理与外部工具，而记忆系统让 DRA 能够利用历史经验——本文系统梳理了 DRA 记忆系统的设计空间。

核心思路：Deep Research Agents 结合了 LLM 的推理能力与外部工具调用，但如何在长期任务中保持一致性和效率仍是挑战。本文系统分析了记忆系统对 DRA 的必要性：从情景记忆（近期交互记录）到语义记忆（任务相关知识）再到工作记忆（当前状态维护），构建完整记忆层。Memory Intelligence Agent 框架的核心创新在于记忆的动态检索与更新机制——根据任务上下文主动选择调用哪一层记忆，而非被动依赖全部历史。实验表明，带记忆的 DRA 在长程任务中显著优于无记忆基线。
启发：记忆不是越多越好——关键在于何时检索哪一层。这对构建真正"懂上下文"的 Agent 系统有直接指导意义。

arXiv 2604.04503
MemMachine: A Ground-Truth-Preserving Memory System for LLM Agents

一句话：提出保持事实连续性的持久记忆系统——解决 LLM Agent 在长程交互中遗忘关键事实、丧失个性化的核心问题。

核心思路：LLM Agent 需要持久记忆来维持个性化、事实连续性和长程任务表现。MemMachine 的核心创新在于"事实保真"机制：不是简单存储历史，而是主动识别并固化关键事实，防止在后续推理中被模型自己产生的噪声覆盖。设计包含记忆压缩（保留高价值信息，丢弃冗余）和记忆检索（基于当前任务动态组装记忆上下文）两个关键模块。实验覆盖个性化问答、多轮对话和长程代码生成等场景，验证了记忆系统在事实准确性和任务完成率上的双重提升。
启发："记忆保真"这个角度很值得重视——模型自己生成的内容有时会"覆盖"原始事实，如何区分并保护关键信息是持久记忆设计的关键。

arXiv 2604.04853
Multi-Layered Memory Architectures for LLM Agents

一句话：LOCOMO 三层记忆架构——情景/语义/工作记忆分层设计，在长上下文 benchmark 上达 46.85% 成功率，显著超越基线。

核心思路：长上下文任务中，LLM 容易"遗忘"早期关键信息。LOCOMO 引入三层记忆：(1) Episodic Memory 记录最近交互，(2) Semantic Memory 存储任务相关的长期知识，(3) Working Memory 维护当前任务状态。三层之间通过注意力机制动态交互，让模型在不同任务阶段调用不同层次的记忆。实验在 LOCOMO、LOCCO、LoCoMo 三个 benchmark 上验证了分层设计的有效性。
启发：长上下文模型的"大海捞针"能力可能不等于有效利用——需要显式的记忆管理机制来指导模型何时检索什么层次的信息。

arXiv 2603.29194
Beyond the AI Tutor: Social Learning with LLM Agents

一句话：探索多 LLM Agent 的社交学习模式——双角色 Agent（GPT-5.2 + Claude Opus 4.6）协同写作，验证社交学习对创意和议论文质量的提升。

核心思路：将 LLM Agent 从"AI 导师"扩展到"社交学习伙伴"。实验设计：参与者分别完成无 AI 辅助、单一 LLM 辅助、以及双角色 LLM 协同写作三种条件。结果显示，双角色 LLM 协作（两个不同模型同时参与，模拟社交学习场景）对创意写作和议论文质量有显著提升，超越单一 LLM 和无辅助基线。研究者认为关键在于不同模型带来了互补的推理视角。
启发：不同能力级别的模型协作是否比同质模型更有效？这对多 Agent 系统设计有启发——也许差异化比同质规模化更重要。

arXiv 2604.02677

多智能体协作与进化

An Empirical Study of Multi-Agent Collaboration for Automated Code Generation

一句话：实证评估多智能体协作编程——supe-agent 模式优于单 Agent，带全局记忆的多智能体在 Git worktree 隔离环境下显著领先。

核心思路：基于严格控制的执行测试平台（Git worktree 隔离 + 显式全局记忆），对比单 Agent 基线与两种多智能体范式：(1) supervisor-agent 模式（一个主 Agent 协调多个子 Agent），(2) 协作多 Agent 模式（多个平等 Agent 分工协作）。实验结果表明，带全局记忆的多智能体在代码生成的正确性、可维护性和长程依赖处理上均显著优于单 Agent。关键发现：全局记忆避免了多 Agent 间的信息丢失，而 supervisor 模式在任务分解明确时效果最佳。
启发：多 Agent 协作中，信息共享机制（全局记忆）与任务分配策略同样重要——不是"越多 Agent 越好"。

arXiv 2603.29632
Evolving Diverse Complex Agents Under Tight Evaluation Budgets

一句话：在有限评估预算下进化多样化复杂 Agent——GEPA 和 Autoresearch 证明 LLMs 可引导 Agentic 工件的进化，即使评估资源紧张。

核心思路：2026 年 LLM 引导的 Agent 进化引发热潮，GEPA 和 Autoresearch 等系统展示了 LLMs 在有限评估预算下生成多样化高性能 Agent 的潜力。本文研究核心挑战：在评估成本高（每个候选 Agent 需要大量测试）的约束下，如何高效进化。提出的方法通过主动学习选择最具信息量的测试用例，减少评估次数的同时保持进化效果。实验在多个 Agent 任务上验证了效率提升。
启发：评估成本是 Agent 进化的关键瓶颈——本文的主动评估策略对实际部署有直接价值，也呼应了"极简评估"的思路。

arXiv 2604.04347
Co-Evolution of Policy and Internal Reward for Language Agents

一句话：策略与内部奖励共进化训练——RLAnything 等系统探索环境、策略和奖励模型在闭环中联合优化的可能性。

核心思路：单独优化策略容易陷入局部最优，RLAnything 提出环境、策略和奖励模型的联合闭环优化方案。核心思想：在 Language Agent 训练中，内部奖励（由模型自己生成）不是固定的，而是随策略共同进化。实验表明，共进化机制让 Agent 能自适应任务难度变化，在开放式环境中表现更稳健。
启发：奖励设计是 Agent 训练的核心问题——固定奖励 vs 可学习奖励的权衡，这篇工作提供了一个有前景的方向。

arXiv 2604.03098
EvoSkills: Self-Evolving Agent Skills via Co-Evolutionary Verification

一句话：通过共进化验证实现 Agent 技能自进化——探索 LLMs 在少样本情况下自主提升专业技能的能力边界。

核心思路：EvoSkills 研究 LLMs 在极其有限标注数据（少样本）下，通过自进化提升专业技能的可能性。核心机制：共进化验证——Agent 在尝试任务的同时生成验证信号，反过来指导自身改进。附录中的系外行星凌星周期检测案例展示了这一方法在科学任务中的应用潜力。关键发现：在某些领域，自进化 Agent 可在零外部标注的情况下达到接近全监督的性能。
启发：自进化 + 共进化验证的组合是一个值得关注的方向，特别是对数据稀缺的领域。

arXiv 2604.01687
ASI-Evolve: AI Accelerates AI

一句话：AI 加速 AI——用 LLM 驱动的自动化探索模型架构设计、科学假设发现等高价值任务，展示 ASI 对 AI 本身的加速效应。

核心思路：ASI-Evolve 探索 LLM 在科学发现中的元能力——不仅用 LLM 做任务，更用 LLM 来改进 LLM 本身。四个主任务：模型架构设计（让 LLM 搜索更好的 Transformer 变体）、科学假设发现（自动生成并验证科学假说）、代码优化（LLM 改进 LLM 生成的代码）和数学推理（自我修正）。实验结果表明，ASI 驱动的探索在效率上显著优于人工设计基线。
启发：这是一个自我改进的闭环——虽然目前阶段仍有局限，但方向上展示了"AI 造 AI"的雏形。

arXiv 2603.29640
Autonomous Agents for Scientific Discovery: Orchestrating Scientists, Language, Code, and Physics

一句话：自主智能体驱动的科学发现——协调科学家、语言、代码和物理模型，自动从数据中提取知识并生成可验证假说。

核心思路：本文提出一个完整的科学发现自动化框架：多个专用 Agent 分别负责知识提取（从文献和数据库）、假说生成（语言模型）、计算验证（代码执行）和物理建模，各 Agent 通过全局协调器共享中间结果。关键创新在于知识提取模块（Section 3.1）——从科学文献中自动提取可机读的假设和证据，形成结构化知识图谱，为后续假说生成提供原料。
启发：多 Agent 协作的价值不仅在于分工，更在于每个 Agent 能专注自身擅长领域的同时，通过全局协调实现超越单个 Agent 的发现能力。
