策略级自修复

Polaris: A Gödel Agent Framework for Small Language Models through Experience-Abstracted Policy Repair

一句话：7B 模型实现递归自我改进——把失败经验抽象成策略补丁，直接修补 Agent 策略代码，而非调参或改 prompt。

核心思路：Gödel Agent 的核心是"自我检查 → 发现错误 → 修补策略"的循环。Polaris 将这个循环落地到 7B 小模型：失败轨迹经"分析 → 策略形成 → 经验抽象 → 代码修补"四步，生成可审计的最小策略补丁。补丁持久化在策略文件中，可跨实例复用。在 MGSM、DROP、GPQA、LitBench 上均获一致提升。
启发：Agent 自改进从"改 prompt / 调参数"升级到"改策略代码"——这更接近人类程序员的 bugfix 逻辑，且补丁可审查。

可迁移记忆系统

MemAPO: Generalizable Self-Evolving Memory for Automatic Prompt Optimization

一句话：用双记忆机制（策略模板 + 错误模式）替代逐任务 prompt 搜索，让 prompt 优化变成可累积的经验积累。

核心思路：传统 prompt 优化每次从头搜。MemAPO 维护双层记忆：正向记忆提炼成功推理轨迹为可复用策略模板，负向记忆组织常见错误为结构化失败模式。新任务到来时同时检索两种记忆，组合生成避免已知错误、促进有效推理的 prompt。通过迭代自反思持续更新记忆库。
启发：把"每次搜索 prompt"变成"积累可迁移的 prompt 经验"——从 O(n) 变成 O(1) 检索 + O(1) 组装，成本大幅下降。

arXiv 2603.21520
MemCollab: Cross-Agent Memory Collaboration via Contrastive Trajectory Distillation

一句话：不同 LLM Agent 之间共享记忆——通过对比推理轨迹提炼"任务级不变量"，剥离 Agent 特有偏差，构建跨模型通用记忆。

核心思路：现有 Agent 记忆与特定模型绑定，跨模型迁移会退化。MemCollab 让不同 Agent 对同一任务生成推理轨迹，通过对比学习提取共有的任务约束（不变量），过滤 Agent 特有偏差。加上任务感知检索机制，确保推理时只激活相关记忆。在数学推理和代码生成上均提升跨模型效果。
启发：Agent 记忆从"个人笔记"升级为"公共知识库"——多 Agent 协作场景的核心基础设施。

进化搜索与 RL 加速

AVO: Agentic Variation Operators for Autonomous Evolutionary Search

一句话：用 Agent 替代经典进化算法的变异/交叉算子——Agent 自主查阅谱系、知识库和执行反馈，在 NVIDIA B200 上进化出超越 cuDNN 的 Attention 内核。

核心思路：传统进化搜索的变异算子是固定的（随机扰动、交叉）。AVO 把变异算子升级为 Agent：自主分析当前解的谱系，查阅领域知识库，执行代码并根据反馈提出、修复、批评、验证编辑。7 天连续进化 multi-head attention，发现的内核比 cuDNN 快 3.5%、比 FlashAttention-4 快 10.5%。GQA 仅需 30 分钟微调即获得 7.0% 提升。
启发：Agent 从"候选解生成器"升级为"进化算子本身"——这打开了一扇门：任何需要探索性优化的工程问题都可以用 Agent 驱动进化。

arXiv 2603.24517
SortedRL: Accelerating RL Training for LLMs through Online Length-Aware Scheduling

一句话：按输出长度排序 rollout 样本，用"长度感知调度"把 LLM 的 RL 训练泡沫比减少 50%+，同数据量性能提升 3.9%~18.4%。

核心思路：RL 训练的 rollout 阶段可占总时间 70%。SortedRL 按输出长度重排样本，短样本优先形成组以尽早触发策略更新，同时支持大 rollout batch、灵活 update batch 和近在线课程学习。引入缓存机制控制 off-policy 程度。在 LLaMA-3.1-8B 和 Qwen-2.5-32B 上均显著加速。
启发：看似简单"排序"，实则是将 curriculum learning 嵌入 rollout 调度。工程优化的典型——算法层面没有大改，系统层面大幅提速。
