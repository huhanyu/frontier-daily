自演化 Agent 架构

A Blueprint for Self-Evolving Coding Agents in Vehicle Aerodynamic Design

一句话：将程序本身（而非静态模型）作为搜索空间，用约束优化 + 岛模型进化来自动发现车辆气动设计的代理模型。

核心思路：不训练一个固定的 surrogate，而是把 surrogate 发现视为程序空间上的约束优化。结合 evaluator 反馈与基于种群的岛演化策略，Agent 在编码-评估-变异循环中不断改进自身的气动仿真代理。
启发：把"模型选择"变成"模型生成+进化"，为工程优化领域提供了 Agent 自演化的新范式。

arXiv 2603.21698
SEVerA: Verified Synthesis of Self-Evolving Agents

一句话：用形式化验证（Dafny）保证自演化 Agent 的每次迭代都满足安全规约——演化可以快，但不能越界。

核心思路：Agent 自演化最大的担忧是"越改越差"或违反安全约束。SEVerA 在每次 Agent 合成/修改自身逻辑后，自动生成 Dafny 验证条件并证明其满足预设规约。不通过验证的变异直接丢弃。
启发：为自演化 Agent 落地提供了"安全网"——形式化保证 > 经验启发式，这在高可靠性场景（航空、医疗、金融）尤其关键。

arXiv 2603.25111
Self-Evolving Multi-Agent Framework for Efficient Decision Making

一句话：多 Agent 系统通过拓扑维度约简实现自演化——从高维特征中自动提炼出精简的决策结构。

核心思路：LLM-based Agent 做实时决策时面临高维特征瓶颈。本框架让多个 Agent 协作的同时，通过自适应结构化特征精炼和拓扑降维，持续演化自身的决策表示。
启发："自演化"不仅是改 prompt / 改策略，也可以是演化内部表示的结构——这对多 Agent 实时协作场景很有价值。

arXiv 2603.23875
EvoIdeator: Evolving Scientific Ideas through Checklist-Grounded Feedback

一句话：用"清单驱动反馈"替代纯 prompt 式批评，在生成-评估循环中持续进化科学假设的质量。

核心思路：现有语言反馈方法多局限于推理时的单轮 prompt 批评。EvoIdeator 将反馈锚定在结构化 checklist 上，使 Agent 的每次变异都可被逐项评估和定向改进，实现了更可控的 idea 进化。
启发：checklist 作为"适应度函数的骨架"，比自由文本反馈更稳定、更可追溯。

arXiv 2603.21728

自改进学习机制

From Solving to Evolving via Retrospective Dual Intrinsic Feedback (RetroAgent)

一句话：借鉴人类"复盘"机制，RetroAgent 通过在线 RL 训练 Agent 不仅完成任务，更在过程中持续演化自身策略。

核心思路：传统 RL Agent 只关心"任务是否完成"。RetroAgent 引入回顾式双重内在反馈：任务完成后自动复盘决策轨迹，提取"哪里做得好"和"哪里可以改进"两种信号，用在线 RL 更新策略。
启发："做完 → 复盘 → 改策略"的闭环，比单纯奖励信号密集得多。类比人类经验学习中"回顾"对成长的关键作用。

arXiv 2603.08561
Experiential Reflective Learning for Self-Improving LLM Agents

一句话：Agent 从过往任务中提取"启发式经验"并注入上下文，实现跨任务的知识迁移与自改进。

核心思路：每完成一个任务，Agent 自动提炼可复用的启发式规则（heuristics）。面对新任务时，根据语义相似度检索相关经验注入 prompt，让过去的教训直接指导当前执行。
启发：与 RetroAgent 互补——一个侧重"过程中的反馈"，一个侧重"跨任务的经验库"。两者结合有望形成更完整的自改进闭环。

arXiv 2603.24639

记忆与经验复用

GSEM: Graph-based Self-Evolving Memory for Experience Consolidation

一句话：用双层图结构组织临床经验——单次决策内的逻辑链 + 跨病例的迁移模式，让 Agent 在持续实践中积累可复用的"经验图谱"。

核心思路：将临床经验组织为双层记忆图：第一层捕获单次诊疗的决策结构（步骤间的依赖关系），第二层捕获跨病例的可迁移模式。Agent 面对新病例时自动检索图中相似路径。
启发：图结构比扁平化的"经验列表"更擅长表达结构化知识和迁移关系。这个框架不限于医疗——任何需要"结构化经验积累"的 Agent 场景都适用。
